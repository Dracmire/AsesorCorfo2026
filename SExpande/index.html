Qué cubre esta presentación

  • Qué es Semilla Expande y cuál es su lógica: cofinanciar crecimiento comercial medible.
  • Qué espera el instrumento: resultados, nuevo mercado, ejecución y control.
  • Cómo se arma un proyecto sólido: brecha → plan → hitos → KPIs → presupuesto.
  • Fricciones típicas (por qué se caen postulaciones) y cómo evitarlas sin “inventar” negocio.
Nota: todo el contenido es neutral. Inserta tu caso en los placeholders: [Producto], [Segmento], [Región], [Canal], [Ticket], [Ciclo], etc.

Checklist rápido (lo que no puede faltar)

  • Línea base: ventas y capacidades actuales (aunque sea estimada).
  • Meta de crecimiento + mecanismo (embudo y capacidad).
  • Nuevo mercado: región o internacional con plan de entrada.
  • Presupuesto con causalidad (cada ítem mueve un KPI).
  • Riesgos + mitigación + reporte mensual.

Sugerencia: usa esta landing como guion de reunión con cliente (30–45 min).

Qué es Semilla Expande (en términos operativos)

Definición de trabajo

  • Instrumento de cofinanciamiento para emprendimientos con validación inicial que necesitan ejecutar un plan de despegue comercial.
  • Corfo “compra” capacidad de crecer: evidencia + gestión + ejecución, no solo “un producto”.
  • Proyecto debe ser un motor de ventas, no un proyecto interno de mejora sin salida comercial.
Traducción: cada actividad y gasto debe explicar cómo contribuye a pipeline, cierres o expansión.

Estructura del instrumento

  • Etapa 1 (ejecución principal): foco en habilitadores + ventas + nuevo mercado.
  • Etapa 2 (extensión): se justifica si ya hay tracción y el cuello está en escala.
  • Cofinanciamiento: el beneficiario aporta un % (propio) y Corfo cubre el resto.

Ajusta montos/condiciones exactas según bases vigentes del llamado que uses.

Resultados esperados (cómo se ve “éxito” en Semilla Expande)

Resultados “duros” (que deben quedar explícitos)

+30% Crecimiento de ventas vs período base (referencia típica)
$6MM+ Incremento mínimo de ventas (referencia típica)
Nuevo mercado Otra región y/o internacional (venta real o primer cierre)
  • Importante: no basta declarar metas. Hay que mostrar mecanismo: embudo, capacidad y actividades.
  • El evaluador busca coherencia: metas ↔ actividades ↔ KPIs ↔ presupuesto.
  • Si el proyecto no “mueve ventas”, normalmente se percibe como desalineado.

Fricciones típicas (dónde se caen postulaciones)

Fricciones de admisibilidad / forma

  • Documentación incompleta o inconsistente (SII/antecedentes/formatos).
  • Datos base débiles: sin línea base de ventas, sin métricas mínimas.
  • Relato poco verificable: no hay evidencia de tracción o aprendizaje.
Mitigación: checklist temprano + consolidación de antecedentes + cuadro “línea base” simple.

Fricciones de fondo (evaluación)

  • Proyecto “de compras”: mucho equipamiento / poco crecimiento.
  • Metas sin mecanismo: +30% sin embudo ni capacidad de venta/entrega.
  • Presupuesto sin causalidad: ítems que no mueven KPIs.
  • Riesgos omitidos: dependencia de un canal/cliente, ciclo largo, operación no preparada.

Cómo se arma un proyecto “ganable” (plantilla neutral)

Estructura recomendada

  • Brecha: ¿qué impide crecer hoy? (comercial / operación / producto)
  • Estrategia: ¿cuál es el motor? (canal / segmento / expansión / oferta)
  • Plan: actividades → entregables → hitos → KPIs (mensual)
  • Presupuesto: cada gasto amarrado a un hito de ventas o expansión
  • Control: reporte mensual + gestión de desviaciones

Regla de oro (para cada ítem)

  • ¿Qué hito comercial habilita?
  • ¿Qué KPI mueve (pipeline/cierre/retención/nuevo mercado)?
  • ¿Qué evidencia deja (documento, registro, contrato, reporte)?
  • ¿Qué pasa si se recorta? (¿se cae el crecimiento o solo “se ve bien”?)

Tip: si un gasto no responde estas preguntas, suele ser un “punto débil” ante evaluación.

Crecimiento comercial (cómo se presenta sin “inventar” negocio)

Lo mínimo que debe verse (con placeholders)

  • Embudo: Leads → Reunión → Propuesta/Piloto → Cierre → Renovación/Expansión.
  • Supuestos cuantitativos: tasas de conversión esperadas, ticket [CLP], ciclo [días], # cuentas/mes.
  • Capacidad: quién vende, quién entrega, cuántos clientes simultáneos soporta la operación.
  • Nuevo mercado: [Región/País] + canal de entrada [Partner/Directo/Distribución] + primer hito (primer contrato/venta).
Plantilla de meta mensual: “Mes N: pipeline $X, cierres $Y, 1er paso nuevo mercado: [actividad verificable]”.

Plan de ejecución tipo (Etapa 1: 9 meses)

Fase 1 — Habilitadores (Mes 1–2)

  • Oferta empaquetada + pricing (o ajuste) + material de ventas.
  • Pipeline inicial: prospección y primeras oportunidades.
  • Hito: “primeras propuestas/pilotos emitidos” + KPI de conversión inicial.

Fase 2 — Conversión (Mes 3–5)

  • Pilotos → cierres (contratos) + optimización de conversión.
  • Operación: onboarding/entrega, soporte, procesos y métricas.
  • Hito: “cierres mensuales sostenidos” + evidencia contractual.

Fase 3 — Expansión (Mes 6–9)

  • Entrada a nuevo mercado: agenda, canal y primera venta/cierre.
  • Escalamiento: estandarización + playbook + canales.
  • Hito: “primer cierre en nuevo mercado” + crecimiento acumulado.

Reporte mensual (control)

  • Avance de hitos + KPIs + desviaciones.
  • Acciones correctivas y cambios controlados.
  • Trazabilidad gasto ↔ actividad ↔ indicador.

Cierre: mensaje clave para Semilla Expande

Qué debe quedar instalado en el cliente

  • Semilla Expande se gana cuando el proyecto demuestra crecimiento medible, con mecanismo y control.
  • La narrativa debe ser “ejecutable”: actividades financiables + KPIs + evidencia.
  • El presupuesto se defiende con causalidad: cada ítem conecta a ventas o a nuevo mercado.
CTA: “Hagamos un cuadro de línea base + metas mensuales + embudo + presupuesto por hito”.

Próximo paso (plantilla)

  • 1) Consolidar admisibilidad/documentos.
  • 2) Definir línea base + meta de crecimiento + nuevo mercado.
  • 3) Diseñar plan de trabajo (9 meses) con KPIs mensuales.
  • 4) Armar presupuesto con causalidad + evidencias.
  • 5) Revisión final: coherencia + riesgos + plan de control.
